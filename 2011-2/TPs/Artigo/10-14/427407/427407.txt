02.) Relacionar os principais critérios usados
     no texto para definir a escolha de uma 
     linguagem para especificação de hardware.

A decisão de qual linguagem escolher é baseada em requisitos básicos.
O autor cita o caso de um usuário iniciante, que ainda não teve muitos
contatos com HDLs e escolhe uma linguagem para executar seu primeiro
projeto. Para a maioria, o usuário deve sempre buscar HDLs que tragam
melhorias para o seu projeto, embora na maior parte dos casos isso não
ocorra logo de início.
Retomando os requisitos básicos citados anteriormente, de acordo com o
autor, o fator mais importante é haver uma razão boa e sólida para a
implementação de uma metodologia de projeto HDL do contrário o design e
talvez até mesmo a empresa pode ser afetada negativamente. Outra razão
para se adotar um HDL é obter uma maior produtividade, mas se não for
possível prever ganhos nessa produtividade, ou se uma perda é consideravelmente
provável, deve-se ponderar os prós e os contras e continuar tentando
implementar um HDL, pois se não houverem testes, nunca se chegará a uma
conclusão aceitável.
Por fim, sempre é importante ficar atento as incompatibilidades e lembrar
do motivo pelo qual foi adotado um HDL e seus benefícios.

03.) Sob que aspectos deverá ser considerado
     o critério "facilidade de uso"?

O critério facilidade de uso pode ser dividido em três categorias, onde
a primeira contém aspectos sobre facilidade de aprendizado, onde fica
claro que é fácil aprender a língua sem experiência previa com HDLs e 
como é simples converter um design com experiência em outra língua para
realizar projetos em HDL.
A segunda categoria que descreve a facilidade de uso, afirma que assim
que o usuário aprende a linguagem, se torna cada vez mais fácil utilizar
recursos mais específicos e avançados.
Já a terceira e última categoria, abrange aspectos sobre usabilidade futura,
onde é dito que do mesmo jeito que a linguagem pode ser suficiente para as
exigências de hoje, poderá também continuar sendo no futuro, mostrando o
potencial de usabilidade a longo prazo.

04.) Sintetizar as principais conclusões.
     em um parágrafo.

Concluindo, após a leitura do artigo é possível dizer que um HDL pode trazer
inúmeros benefícios para um design ou até mesmo para uma empresa, desde
que sejam cuidadosamente avaliados os prós e os contras de se implantar essa
linguagem. Depois de feito isso, também é mostrado a flexibilidade de um HDL,
onde é afirmado que para um usuário iniciante é muito fácil de aprender e para
um usuário  que já tenha algum conhecimento prévio de alguma outra linguagem,
isso se torna mais fácil ainda e além disso, depois de aprendido o básico, o
usuário fica a um passo de utilizar os recursos avançados da ferramenta. Por
fim, é apresentado a perspectiva futura de um HDL, onde é mostrado que a linguagem
do mesmo jeito que atende as exigências atuais, poderá continuar atendendo as
exigências futuras.
